# Клиентский сегмент как состояние восприимчивости к обмену

## Ключевое определение

> **Клиентский сегмент** – это группа людей, объединённых общим **состоянием восприимчивости к обмену**: набором длящихся характеристик, при которых у них с высокой вероятностью возникает мотивационный конфликт, разрешаемый данным артефактом.

## Фундаментальное различие: событие vs состояние

### Событие (неправильно для сегментации)

**Событие** – это точка во времени, момент, который произошёл и прошёл.

**Примеры событий:**
- "Купил здоровое питание"
- "Заказывал доставку еды"
- "Посещал тренинг"
- "Брал кредит"

**Характеристики:**
- Относится к прошлому
- Не гарантирует текущей актуальности
- Невозможно предсказать будущее поведение
- Требует доступа к истории действий

### Состояние (правильно для сегментации)

**Состояние** – это характеристика, которая существует в течение некоторого периода времени.

**Примеры состояний:**
- "Является родителем ребёнка 0-2 лет"
- "Работает удалённо"
- "Регулярно занимается спортом (3+ раза/неделю)"
- "Имеет действующий ипотечный кредит"

**Характеристики:**
- Относится к настоящему
- Актуально здесь и сейчас
- Позволяет прогнозировать поведение
- Можно идентифицировать по наблюдаемым признакам

## Тест на длящийся признак

### Формула проверки

Подставьте в предложение **"ЯВЛЯЕТСЯ"** вместо "купил/покупал":

❌ **"Человек ЯВЛЯЕТСЯ купившим здоровое питание"**
- Звучит неестественно
- Это событие, а не состояние

✅ **"Человек ЯВЛЯЕТСЯ родителем малыша"**
- Звучит естественно
- Это состояние

✅ **"Человек ЯВЛЯЕТСЯ менеджером в IT"**
- Звучит естественно
- Это состояние

### Три вопроса для проверки

**1. Это про человека СЕЙЧАС или про его прошлое?**
- Сейчас → состояние ✅
- Прошлое → событие ❌

**2. Это ЯВЛЯЕТСЯ частью его идентичности или БЫЛО действием?**
- Является → состояние ✅
- Было → событие ❌

**3. Это будет актуально завтра/через неделю/через месяц?**
- Да → длящийся признак ✅
- Неизвестно → скорее событие ❌

## Что такое "восприимчивость к обмену"

### Определение

**Состояние восприимчивости к обмену** – это такая конфигурация характеристик человека, при которой у него с высокой вероятностью возникнет мотивационный конфликт, разрешаемый данным артефактом.

### Логическая цепочка
```
СОСТОЯНИЕ ВОСПРИИМЧИВОСТИ
    ↓ (коррелирует с)
ТИПИЧНАЯ СИТУАЦИЯ
    ↓ (порождает)
МОТИВАЦИОННЫЙ КОНФЛИКТ
    ↓ (разрешается)
АРТЕФАКТОМ
```

### Пример

**Состояние восприимчивости:**
"Офисные сотрудники 25-40 лет, работающие 50+ часов/неделю, тренирующиеся в зале 3+ раза/неделю"

**Типичная ситуация:**
"Рабочий день + тренировка отнимают всё время"

**МК:**
"Хочу поддерживать форму и питаться правильно, но нет времени готовить + после зала лень"

**Артефакт:**
"Доставка готовых порций с подсчитанными БЖУ под спортивный режим"

## Почему длительность критична

### Предсказательная сила
```
Длительность состояния ↑
    ↓
Окно возможности для обмена ↑
    ↓
Вероятность попасть в нужный момент ↑
    ↓
Эффективность таргетинга ↑
```

### Формула предсказательной силы
```
ПРЕДСКАЗАТЕЛЬНАЯ СИЛА = 
    Вероятность МК × Длительность состояния × Точность идентификации
```

**Где:**
- **Вероятность МК** – насколько вероятно, что у человека в этом состоянии возникнет нужный МК
- **Длительность состояния** – как долго человек будет в этом состоянии (окно для таргетинга)
- **Точность идентификации** – можем ли мы достоверно определить, что человек в этом состоянии

### Сравнение: событие vs состояние

#### ❌ Сегмент через событие
```
"Заказывал здоровую еду раньше"

Вероятность МК: низкая 
  (контекст мог измениться)
  
Длительность: 0 
  (событие в прошлом)
  
Идентификация: только по базе

→ Предсказательная сила: ~0
```

#### ✅ Сегмент через состояние
```
"Офисные сотрудники 25-40 лет, 
тренирующиеся в зале 3+ раза/неделю"

Вероятность МК: высокая
  → регулярно нужно питаться правильно
  → нет времени готовить (офис + зал)
  
Длительность: месяцы/годы
  → работа в офисе – устойчиво
  → регулярные тренировки – устойчивая привычка
  
Идентификация: высокая
  → по должности (LinkedIn, hh.ru)
  → по чекинам в залах (если есть партнёрство)
  → по интересам (таргет в соцсетях)

→ Предсказательная сила: высокая
```

## Градации длительности состояний

### Типология по устойчивости
```
МЕГА-УСТОЙЧИВЫЕ (10+ лет)
→ "Имеет высшее образование"
→ "Является собственником бизнеса"
→ Применение: долгосрочная стратегия

УСТОЙЧИВЫЕ (1-5 лет)
→ "Работает в должности X"
→ "Родитель ребёнка возраста Y"
→ Применение: идеально для большинства B2B и сложных B2C

СРЕДНЕСРОЧНЫЕ (3-12 месяцев)
→ "Проходит адаптацию на новой работе"
→ "Находится в процессе ремонта"
→ Применение: целевые кампании

КРАТКОСРОЧНЫЕ (недели-2 месяца)
→ "Тренируется первый месяц"
→ "Придерживается диеты (без устойчивой привычки)"
→ Применение: требуют быстрой конверсии

СИТУАТИВНЫЕ (дни-недели)
→ "Планирует путешествие"
→ "Ищет подарок"
→ Применение: горячий трафик, импульсные продажи
```

**Правило:** Чем сложнее/дороже артефакт и длиннее цикл продажи, тем устойчивее должно быть состояние.

## Типичные ошибки сегментации

### Ошибка 1: Сегментация через событие

**❌ Неправильно:**
"Покупатели здорового питания"

**Почему не работает:**
- Это событие, а не состояние
- Неизвестна текущая актуальность
- Невозможно идентифицировать без базы
- Нет предсказательной силы

**✅ Правильно:**
"Спортсмены-любители, тренирующиеся 3+ раза в неделю, возраст 25-40"

### Ошибка 2: Подмена МК в критерии сегмента

**❌ Неправильно:**
"Люди, которые хотят здорово питаться"

**Почему не работает:**
- Желание – это часть МК, а не признак сегмента
- Логическая ошибка: сегмент → МК, а не наоборот
- Невозможно проверить заранее

**✅ Правильно:**
"Родители детей 0-3 лет с высшим образованием, проживающие в городах 1млн+"
→ У них с высокой вероятностью возникнет желание здорово питаться (это следствие)

### Ошибка 3: Слишком широкое состояние

**❌ Неправильно:**
"Живут в Москве"

**Почему не работает:**
- Половина города – это не сегмент
- МК непредсказуем без уточнения
- Нет корреляции с восприимчивостью

**✅ Правильно:**
"Живут в Москве в районе Сити, работают в офисе, доход 150к+/мес"

### Ошибка 4: Неидентифицируемое состояние

**❌ Неправильно:**
"Компании, где в совете директоров всерьёз обсуждают автоматизацию"

**Почему не работает:**
- Невозможно проверить по открытым данным
- Неясно, что значит "всерьёз"
- Нет способа найти таких

**✅ Правильно:**
"Промышленные компании 500+ человек, где CIO или CDO в должности менее 6 месяцев"

## Когда события могут быть полезны

### Контекст: работа с существующей базой

Если у вас есть база клиентов с историей действий, **события могут индицировать состояния**.

### Логика использования событий
```
СОБЫТИЯ (из базы)
    ↓
АНАЛИЗ ПАТТЕРНОВ
    ↓
ГИПОТЕЗА О СОСТОЯНИИ
    ↓
СЕГМЕНТАЦИЯ ПО СОСТОЯНИЯМ
    ↓
ПОСТРОЕНИЕ МК
```

### Пример для маркетплейса

**❌ Примитивно:**
```
Сегмент: "Купили органику"
Оффер: "Вот ещё органика!"
```

**✅ Продвинуто:**
```
События: 
- Регулярно покупает органику
- Детское питание 6+ месяцев
- Экотовары для дома
- Паттерн: 2 раза/месяц последние 4 месяца

↓

Вывод о состоянии: 
"Молодая мать ребёнка ~8-10 месяцев,
высокая экологическая осознанность,
доход выше среднего"

↓

Сегмент: "Эко-мамы младенцев"

↓

Оффер: "Новая линейка органической детской косметики"
(специфично под состояние, а не просто "ещё органика")
```

### Правило комбинации

Даже при наличии базы:
```
Устойчивые признаки (кто они в жизни)
    +
Поведенческие маркеры (что делали в продукте)
    =
Сегмент для таргетированного оффера
```

**Пример:**
```
"Женщины 25-40, проживающие в городах 1млн+, 
с высшим образованием (по анкете при регистрации),
покупавшие органическую косметику 3+ раза за полгода,
средний чек 2000+₽"
```

## Чек-лист валидации сегмента (ДЛИД)

### Мнемоника "ДЛИД"

**Д** – **Д**лящееся (не событие)
**Л** – **Л**огически связано с МК
**И** – **И**дентифицируемое
**Д** – **Д**остаточно узкое

Если все 4 пункта выполнены → у вас **состояние восприимчивости к обмену**.

### Развёрнутая проверка

#### 1. Это длящееся состояние?

**Тест:**
```
"Человек ЯВЛЯЕТСЯ этим прямо сейчас 
и будет являться минимум N времени?"
       
✅ Да → идём дальше
❌ Нет → переформулировать
```

#### 2. Состояние коррелирует с вероятностью МК?

**Тест:**
```
"У большинства людей в этом состоянии 
возникает наш МК?"
       
✅ Да (>50%) → идём дальше
❌ Нет → искать более точное состояние
```

#### 3. Состояние достаточно узкое?

**Тест:**
```
"Это описывает специфическую группу людей,
а не половину населения?"
       
✅ Да → идём дальше
❌ Нет → добавить уточняющие критерии
```

#### 4. Состояние можно идентифицировать?

**Тест:**
```
"Можно ли найти людей в этом состоянии
по наблюдаемым признакам?"
       
✅ Да → идём дальше
❌ Нет → пересмотреть критерии
```

#### 5. Длительности достаточно для цикла продажи?

**Тест:**
```
"Состояние длится дольше, чем нужно времени
на привлечение и конверсию?"
       
Примеры:
- Цикл продажи 2 недели, состояние 3 дня → проблема
- Цикл продажи 2 недели, состояние 6 месяцев → ок

✅ Да → сегмент валидный
❌ Нет → искать более устойчивое состояние
```

## Примеры правильной сегментации

### Пример 1: Курсы английского

**❌ Неправильно:**
"Люди, которые покупали курсы английского"

**✅ Правильно:**
```
"Менеджеры уровня senior/lead в международных IT-компаниях,
проработавшие в должности менее 6 месяцев"

Почему работает:
- Длящееся: новая должность = период адаптации (6-12 мес)
- Связано с МК: новая должность в международной компании 
  → нужен английский для коммуникаций
- Идентифицируемое: LinkedIn, hh.ru
- Узкое: специфическая группа
```

### Пример 2: Доставка здоровой еды

**❌ Неправильно:**
"Заказывали здоровую еду раньше"

**✅ Правильно:**
```
"Офисные сотрудники 25-40 лет, работающие 50+ часов/неделю,
тренирующиеся в зале 3+ раза/неделю последние 3+ месяца"

Почему работает:
- Длящееся: работа + устойчивая привычка тренировок 3+ мес (месяцы/годы)
- Связано с МК: нет времени готовить + нужно правильно питаться
- Идентифицируемое: должность + чекины в залах
- Узкое: конкретная группа с предсказуемым образом жизни
```

### Пример 3: HR-аудит

**❌ Неправильно:**
"Компании, которые заказывали HR-аудит"

**✅ Правильно:**
```
"HRD в компаниях 100+ человек, 
работающие в должности менее 6 месяцев"

Почему работает:
- Длящееся: период адаптации на новой должности (6-12 мес)
- Связано с МК: нужно разобраться в процессах + 
  нет документации от предшественника
- Идентифицируемое: LinkedIn, hh.ru, профильные сообщества
- Узкое: конкретная должность в конкретный период
```

## Связь с остальными блоками ДКЦП

### Логическая цепочка модели
```
КЛИЕНТСКИЙ СЕГМЕНТ
(состояние восприимчивости)
    ↓ определяет контекст для
МОТИВАЦИОННЫЙ КОНФЛИКТ
(возникает У ЛЮДЕЙ В ЭТОМ СОСТОЯНИИ)
    ↓ требует разрешения через
АРТЕФАКТ + АТРИБУТЫ
(разрешают МК людей в этом состоянии)
    ↓ подтверждаются
АТРИБУТЫ КОМПАНИИ
(релевантны для людей в этом состоянии)
    ↓ символизируются через
ЦЕННОСТЬ
(значима для людей в этом состоянии)
```

### Пример сходимости

**Сегмент (состояние восприимчивости):**
"HRD в компаниях 100+ человек, работающие в должности менее 6 месяцев"

**МК:**
"Хочу быстро разобраться в текущих HR-процессах и найти точки роста, но предыдущий HRD не оставил внятной документации + боюсь что-то критичное упустить"

**Артефакт:**
"HR-аудит с картированием процессов и roadmap на 90 дней"

**Атрибуты компании:**
"15 лет на рынке, 200+ HR-аудитов, клиенты: Яндекс, Сбер, МТС"

**Ценность:**
"Уверенность, Контроль, Профессионализм"

**Проверка сходимости:**
- ✅ Состояние → МК: новый HRD обязательно столкнётся с хаосом
- ✅ МК → Артефакт: аудит именно про "разобраться быстро"
- ✅ Атрибуты компании → Состояние: опыт работы с крупными компаниями важен для HRD топ-компании
- ✅ Ценность → Состояние: "уверенность" критична для человека в период адаптации

## Практический алгоритм работы

### Для создания нового ЦП

**Шаг 1: Определить артефакт**
Что мы продаём?

**Шаг 2: Определить МК**
Какую проблему решаем?

**Шаг 3: Найти состояние восприимчивости**
```
Вопрос: "Кто эти люди (по длящимся признакам),
у которых с высокой вероятностью возникает этот МК?"
```

**Шаг 4: Проверить через чек-лист ДЛИД**
- Длящееся?
- Логически связано с МК?
- Идентифицируемое?
- Достаточно узкое?

**Шаг 5: Убедиться в практической применимости**
- Можем ли мы идентифицировать людей в этом состоянии?
- Сколько их примерно?
- Как до них дотянуться?
- Длительности хватит на цикл продажи?

### Для анализа существующего ЦП

**Шаг 1: Посмотреть на формулировку сегмента**

**Шаг 2: Задать вопросы:**
- Это состояние или событие?
- Это длящийся признак?
- Коррелирует ли с МК?
- Можно ли идентифицировать?

**Шаг 3: Если проблемы → переформулировать:**
```
От событий → к состояниям
От широких → к узким
От неидентифицируемых → к идентифицируемым
От желаний → к характеристикам
```

## Финальная формула

> **Клиентский сегмент в ДКЦП – это не список тех, кто покупал похожее, и не описание тех, кто "хочет". Это определение СОСТОЯНИЯ ВОСПРИИМЧИВОСТИ К ОБМЕНУ – набора длящихся характеристик человека, при которых у него с высокой вероятностью возникнет МК, разрешаемый нашим артефактом.**

### Три критерия валидности

1. **ДЛЯЩЕЕСЯ** – существует в настоящем и будущем (не событие из прошлого)
2. **ВОСПРИИМЧИВОЕ** – коррелирует с вероятностью МК
3. **ИДЕНТИФИЦИРУЕМОЕ** – можно найти и дотянуться

### Формула запоминания
```
СОСТОЯНИЕ (кто они СЕЙЧАС)
    ×
ВОСПРИИМЧИВОСТЬ (вероятность МК)
    ×
ИДЕНТИФИЦИРУЕМОСТЬ (можем найти)
    =
ВАЛИДНЫЙ СЕГМЕНТ
```
