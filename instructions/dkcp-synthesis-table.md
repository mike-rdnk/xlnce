# Инструкция по заполнению таблицы синтеза атрибутов

## Назначение таблицы синтеза

Таблица синтеза — это инструмент для систематизации атрибутов артефакта с точки зрения потребительского опыта. Она отвечает на вопросы:
- Какой артефакт является носителем атрибута?
- В какой ситуации клиент получает опыт от атрибута?
- Через какую репрезентативную систему клиент воспринимает атрибут?
- Какой конкретный опыт генерирует атрибут?
- Как атрибут связан с разрешением мотивационного конфликта?

## Структура таблицы

| Артефакт-носитель | Атрибут артефакта | Ситуация проявления опыта | Репрезентативная система(ы) | Опыт клиента | Связь с конфликтом |
| ----------------- | ----------------- | ------------------------- | --------------------------- | ------------ | ------------------ |
| ...               | ...               | ...                       | ...                         | ...          | ...                |

## Описание колонок

### 1. Артефакт-носитель

**Что указывать:** Название артефакта или элемента артефакта, которому принадлежит атрибут.

**Правила заполнения:**
- Используйте названия из раздела "Артефакты" вашей ДКЦП-модели
- Для эмерджентных атрибутов укажите "Система в целом" или перечислите артефакты через "+"

### 2. Атрибут артефакта

**Что указывать:** Конкретное свойство артефакта, которое генерирует потребительский опыт.

**Требования к атрибуту:**
1. Разрешает мотивационный конфликт ценностного предложения
2. Может использоваться для сравнения с конкурентами
3. Генерирует потребительский опыт через репрезентативные системы

**Правила формулировки:**
- Формулируйте конкретно и сенсорно
- Избегайте абстракций ("высокое качество", "удобство")
- Атрибут должен быть восринимаем через одну из репрезентативных систем

### 3. Ситуация проявления опыта

**Что указывать:** Конкретный контекст или момент, когда клиент получает опыт от атрибута.

**Вопросы для определения ситуации:**
- Когда клиент сталкивается с этим атрибутом?
- В какой деятельности клиента проявляется атрибут?
- Что должно произойти, чтобы клиент почувствовал атрибут?

**Формат:** Краткое описание ситуации или этапа customer journey.

### 4. Репрезентативная система(ы)

**Что указывать:** Одну или несколько систем восприятия из VAKOG.

| Код | Система                       | Что воспринимает клиент |
| --- | ----------------------------- | ----------------------- |
| V   | Visual (визуальная)           | Видит                   |
| A   | Auditory (аудиальная)         | Слышит                  |
| K   | Kinesthetic (кинестетическая) | Ощущает физически       |
| O   | Olfactory (запаховая)         | Чувствует запах         |
| G   | Gustatory (вкусовая)          | Чувствует вкус          |

**Правила заполнения:**
- Указывайте все релевантные системы через запятую (например: "V, K")
- Большинство атрибутов воспринимаются через 1-2 системы
- Кинестетическая система включает не только тактильные ощущения, но и ощущение времени, усилия, комфорта

### 5. Опыт клиента

**Что указывать:** Конкретное сенсорное переживание клиента от атрибута.

**Два типа генерации опыта:**

| Тип                  | Описание                                   | Формат записи                                           |
| -------------------- | ------------------------------------------ | ------------------------------------------------------- |
| **Появление (+)**    | Клиент начинает ощущать что-то новое       | "(+) начинает [видеть/слышать/ощущать] + [что именно]"  |
| **Исчезновение (−)** | Клиент перестаёт ощущать что-то негативное | "(−) перестаёт [видеть/слышать/ощущать] + [что именно]" |

**Формат описания:**
- Указывайте тип генерации: (+) или (−)
- Используйте глагол "начинает" или "перестаёт"
- Описывайте конкретные сенсорные детали
- Избегайте интерпретаций и оценок

**Примеры:**
- ✅ "(+) начинает видеть структурированную таблицу с 9 блоками и их связями"
- ✅ "(−) перестаёт ощущать неопределённость — видит чёткий план действий"
- ✅ "(+) начинает ощущать мгновенный отклик интерфейса при нажатии"
- ✅ "(−) перестаёт слышать шум кулера при работе"
- ❌ "Понимает логику модели" (это интерпретация, не сенсорный опыт)
- ❌ "Чувствует, что система быстрая" (это оценка, не сенсорный опыт)

### 6. Связь с конфликтом

**Что указывать:** Как атрибут влияет на разрешение мотивационного конфликта.

**Формат:**
- Укажите, на какой элемент конфликта воздействует атрибут (барьер, драйвер, П1П)
- Укажите метод воздействия: устранение, снижение интенсивности, обход, рефрейминг
- При наличии нескольких конфликтов — укажите номер (КМК№1, КМК№2)

## Алгоритм заполнения

### Шаг 1. Подготовьте исходные данные

Соберите из ДКЦП-модели:
- Список артефактов (раздел 3)
- Мотивационный конфликт(ы) с элементами грозовой тучи (раздел 2)
- Существующие формулировки атрибутов (раздел 4)

### Шаг 2. Для каждого атрибута определите ситуацию

Задайте вопрос: "В какой момент взаимодействия с артефактом клиент столкнётся с этим атрибутом?"

### Шаг 3. Определите репрезентативную систему

Задайте вопрос: "Через какой канал восприятия (вижу/слышу/ощущаю/обоняю/пробую на вкус) клиент почувствует этот атрибут?"

Если ответ "никакой" или "только умом" — переформулируйте атрибут или признайте, что это не атрибут.

### Шаг 4. Опишите конкретный опыт

Определите тип генерации опыта:
- **(+) Появление**: клиент начинает ощущать что-то новое
- **(−) Исчезновение**: клиент перестаёт ощущать что-то негативное

Сформулируйте, что именно клиент начнёт или перестанет видеть/слышать/ощущать. Используйте сенсорно-специфичную лексику.

### Шаг 5. Установите связь с конфликтом

Определите, какой элемент конфликта атрибут помогает разрешить и каким методом.

### Шаг 6. Проверьте заполненную строку

Убедитесь, что:
- Атрибут соответствует трём требованиям (разрешает конфликт, годится для сравнения, генерирует опыт)
- Ситуация конкретна и наблюдаема
- Опыт описан сенсорно, а не абстрактно
- Связь с конфликтом логична

## Типичные ошибки

### Ошибка 1: Абстрактные атрибуты

❌ **Неправильно:**
| Артефакт   | Атрибут          | Ситуация | Система | Опыт                | Конфликт |
| ---------- | ---------------- | -------- | ------- | ------------------- | -------- |
| Консалтинг | Высокое качество | Проект   | —       | Доволен результатом | Барьер   |

✅ **Правильно:**
| Артефакт         | Атрибут                                                 | Ситуация         | Система | Опыт                                                      | Конфликт                                    |
| ---------------- | ------------------------------------------------------- | ---------------- | ------- | --------------------------------------------------------- | ------------------------------------------- |
| Отчёт по проекту | 40+ страниц структурированного анализа с визуализациями | Получение отчёта | V       | Видит объёмный документ с графиками, таблицами и выводами | Устранение барьера "нет доказательной базы" |

### Ошибка 2: Смешение атрибута и процесса

❌ **Неправильно:**
| Артефакт   | Атрибут          |
| ---------- | ---------------- |
| Консалтинг | Методология ДКЦП |

✅ **Правильно:**
| Артефакт    | Атрибут                                                   |
| ----------- | --------------------------------------------------------- |
| ДКЦП-модель | Структура из 9 взаимосвязанных блоков в табличном формате |

### Ошибка 3: Опыт описан как интерпретация

❌ **Неправильно:**
| Опыт клиента                      |
| --------------------------------- |
| Чувствует профессионализм команды |

✅ **Правильно:**
| Опыт клиента                                                                            |
| --------------------------------------------------------------------------------------- |
| Слышит структурированные вопросы консультанта; видит подготовленные материалы к встрече |

### Ошибка 4: Отсутствие репрезентативной системы

Если вы не можете определить репрезентативную систему для атрибута, это сигнал:
- Атрибут сформулирован слишком абстрактно
- Это не атрибут, а характеристика процесса или компании
- Нужно декомпозировать атрибут на более конкретные составляющие

## Связь с другими инструментами ДКЦП

- **Диаграмма атрибутов** (`dkcp-attributes-mermaid-template.md`): таблица синтеза является источником данных для визуализации связей атрибутов с конфликтами
- **Таблица мотивационного конфликта**: колонка "Связь с конфликтом" должна соответствовать элементам грозовой тучи
- **Критерии проверки** (`dkcp-checkup-criteria.md`): таблица синтеза помогает проверить, что каждый атрибут действительно разрешает конфликт и генерирует опыт
