# Реверс-инжиниринг ценностного предложения

## Шаг 1. Получить материал

Попроси приложить материал для анализа: PDF-презентацию или HTML-страницу сайта. После получения уточни, есть ли дополнительный контекст.

> Если материал — вакансия или иной носитель EVP, применяй поправку из [[dkcp-evp.md]].

## Шаг 2. Анализ аргументов

1. Выдели все аргументы в материале, отсортируй по значимости
2. Для каждого аргумента: определи, о связи каких блоков ДКЦП он сообщает
3. Восстанови содержимое блоков из найденных аргументов — не придумывай блоки, которых нет

## Шаг 3. Проверка логики

**Сегмент и МК:**
- Нет ли критериев сегментации с элементами МК (пример: «мамы, испытывающие тревожность» — тревожность — это МК, а не признак сегмента)
- Есть ли корреляция между критериями сегментации и вероятностью МК?

**Атрибуты артефакта:**
- Не приняты ли аргументы за атрибуты?
- Корректна ли разбивка на артефакты и атрибуты? (критерии из [[dkcp-artefact-attributes.md]])
- Адресованы ли атрибуты МК? Какие методы разрешения конфликта используются? ([[conflict-resolution-methods.md]])
- Достаточно ли атрибутов для закрытия МК?

**Цена и условия обмена:**
- Не противоречат ли ценовые атрибуты заявленным ценностям? (пример: «скидка 40%» ↔ ценность «статусность»)

## Шаг 4. Формулировка замечаний

Сформулируй претензии к логике ДКЦП, используя [[dkcp-checkup-criteria.md]]. Укажи, в каком блоке нужны изменения, и предложи рекомендации.

## Шаг 5. Вывод результата

Оформи восстановленную модель в Markdown по шаблону [[dkcp-template-full.md]].
