# Инструкции для ИИ-ассистента по использованию материалов xlnce

## Назначение

Этот файл содержит инструкции для ИИ-ассистентов (например, Cursor AI) по использованию материалов репозитория xlnce при работе с ценностными предложениями и методологией ДКЦП.

## Когда использовать материалы xlnce

Материалы из этого репозитория должны использоваться автоматически, когда в диалоге речь заходит о:

- **Ценностном предложении (ЦП)** или Value Proposition
- **ДКЦП** (Дворец Культуры Ценностного Предложения)
- **Мотивационных конфликтах** клиентов
- **Артефактах и атрибутах** решения
- **Условиях обмена** между клиентом и компанией
- **Аргументации** для клиентов
- **Customer Journey Map (CJM)** или Employee Journey Map (EJM)
- **Анатомии потребности**
- **Метамодели языка** для анализа коммуникации
- **Реверс-инжиниринге** существующих решений

## Структура материалов

### `materials/` — Базовые концепции и теория

- **`dkcp-overview.md`** — Обзор методологии ДКЦП, основные принципы
- **`dkcp-evp.md`** — Employee Value Proposition (ценностное предложение для сотрудников)
- **`problem-anatomy.md`** — Анатомия потребности (мотивационный фреймворк)
- **`usp-anatomy.md`** — Анатомия УТП (уникального торгового предложения)
- **`value-logics.md`** — Логики создания ценности

**Когда использовать:** Для понимания базовых концепций, терминологии, философии методологии.

### `instructions/` — Практические инструкции

- **`dkcp-basic-narrative.md`** — Создание базового нарратива из аргументов ДКЦП
- **`dkcp-motivational-conflict-creation.md`** — Создание и формулировка мотивационного конфликта
- **`dkcp-motivational-conflict-verification.md`** — Проверка качества мотивационного конфликта
- **`dkcp-motivational-conflict-resolve.md`** — Разрешение мотивационного конфликта через артефакт
- **`dkcp-customer-segment-creation.md`** — Создание и описание клиентского сегмента
- **`dkcp-artefact-attributes.md`** — Работа с артефактами и их атрибутами
- **`dkcp-values.md`** — Работа с ценностями в модели
- **`dkcp-checkup-criteria.md`** — Критерии проверки и сходимости ДКЦП-модели
- **`dkcp-reverse-engeneering.md`** — Реверс-инжиниринг существующих решений
- **`dkcp-user-briefing.md`** — Брифинг клиента для создания ДКЦП
- **`dkcp-local-conflicts.md`** — Работа с локальными конфликтами
- **`dkcp-iqrt-instruction.md`** — Инструкция по IQRT (таблица мотивационного конфликта)
- **`metamodel analysis.md`** — Метамодельный анализ коммуникации

**Когда использовать:** При выполнении конкретных задач по созданию или анализу ДКЦП-модели.

### `templates/` — Шаблоны для работы

- **`dkcp-template-full.md`** — Полный шаблон ДКЦП-модели
- **`dkcp-template-basic.md`** — Базовый шаблон ДКЦП
- **`dkcp-motivational-conflict-table-template.md`** — Шаблон таблицы мотивационного конфликта
- **`dkcp-motivational-conflict-mermaid-template.md`** — Шаблон диаграммы конфликта (Mermaid)
- **`dkcp-attributes-mermaid-template.md`** — Шаблон диаграммы атрибутов
- **`dkcp-exchange-sequence-template.md`** — Шаблон последовательности обмена
- **`dkcp-checkup-template.md`** — Шаблон для проверки модели
- **`dkcp-html-template.md`** — HTML-шаблон для презентации

**Когда использовать:** При создании новых моделей или документов по ДКЦП.

## Алгоритм работы с материалами

### 1. Определение задачи

Когда пользователь задаёт вопрос или просит помочь с ЦП/ДКЦП:

1. Определи тип задачи (создание модели, анализ, проверка, реверс-инжиниринг)
2. Найди соответствующую инструкцию в `instructions/`
3. Используй семантический поиск для нахождения релевантных материалов

### 2. Использование материалов

**Всегда:**
- Начинай с чтения релевантных инструкций из `instructions/`
- Используй терминологию из `materials/dkcp-overview.md`
- Ссылайся на конкретные файлы, которые используешь
- Следуй структуре и критериям из инструкций

**Пример:**
```
При создании мотивационного конфликта используй инструкцию 
`instructions/dkcp-motivational-conflict-creation.md` и проверяй 
результат по критериям из `instructions/dkcp-checkup-criteria.md`.
```

### 3. Проверка результата

После создания или анализа модели:
- Проверь соответствие критериям из `instructions/dkcp-checkup-criteria.md`
- Убедись, что все обязательные элементы присутствуют
- Проверь терминологию на соответствие методологии

## Ключевые принципы ДКЦП

1. **ДКЦП — это язык описания**, а не шаблон для заполнения
2. **ЦП — информационная структура** (9 взаимосвязанных блоков), а не продукт или текст
3. **Всегда проверяй сходимость** по формальным критериям
4. **Используй метамодель** для выявления когнитивных искажений в коммуникации
5. **Мотивационный конфликт** — центральный элемент модели

## Структура ДКЦП (9 блоков)

| №   | Блок                   | Описание                                          |
| --- | ---------------------- | ------------------------------------------------- |
| 1   | Клиентский сегмент     | Кто является клиентом                             |
| 2   | Мотивационный конфликт | Проблема клиента (П1П)                            |
| 3   | Артефакт               | Что предлагается клиенту                          |
| 4   | Атрибуты артефакта     | Характеристики артефакта                          |
| 5   | Вклад клиента          | Что отдаёт клиент (материальное и нематериальное) |
| 6   | Условия обмена         | Как происходит обмен                              |
| 7   | Атрибуты компании      | Характеристики компании                           |
| 8   | Ценность               | Символический смысл сделки для клиента и компании |
| 9   | Аргументы              | Объяснение связи между блоками                    |

## Поиск материалов

Используй семантический поиск в директории `_shared/public/xlnce/` для нахождения релевантных инструкций по ключевым словам:

- "мотивационный конфликт" → `dkcp-motivational-conflict-*.md`
- "клиентский сегмент" → `dkcp-customer-segment-creation.md`
- "артефакт" → `dkcp-artefact-attributes.md`
- "проверка" или "сходимость" → `dkcp-checkup-criteria.md`
- "нарратив" → `dkcp-basic-narrative.md`
- "реверс" → `dkcp-reverse-engeneering.md`

## Интеграция с Cursor Rules

В репозитории `okb-pndlnk` создано правило `.cursor/rules/dkcp-xlnce.mdc`, которое автоматически активируется при работе с темами ДКЦП и ЦП.

Это правило указывает на материалы в `_shared/public/xlnce/` и обеспечивает их использование в диалогах.
