# Шаблон таблицы синтеза атрибутов

## Назначение

Этот шаблон предназначен для систематизации атрибутов артефакта с точки зрения потребительского опыта и связи с мотивационным конфликтом.

Подробная инструкция по заполнению: [[dkcp-synthesis-table]]

## Справочник: Типы генерации опыта

| Тип                  | Описание                                   | Формат записи                    | Пример                                      |
| -------------------- | ------------------------------------------ | -------------------------------- | ------------------------------------------- |
| **(+) Появление**    | Клиент начинает ощущать что-то новое       | "(+) начинает [глагол] + [что]"  | (+) начинает видеть структурированный отчёт |
| **(−) Исчезновение** | Клиент перестаёт ощущать что-то негативное | "(−) перестаёт [глагол] + [что]" | (−) перестаёт слышать шум при работе        |

## Справочник: Репрезентативные системы (VAKOG)

| Код   | Система                       | Что воспринимает клиент | Примеры                                      |
| ----- | ----------------------------- | ----------------------- | -------------------------------------------- |
| **V** | Visual (визуальная)           | Видит                   | Цвет, форма, интерфейс, графики, документы   |
| **A** | Auditory (аудиальная)         | Слышит                  | Звук, голос, тон, тишина, музыка             |
| **K** | Kinesthetic (кинестетическая) | Ощущает физически       | Вес, температура, текстура, скорость, усилие |
| **O** | Olfactory (запаховая)         | Чувствует запах         | Аромат, свежесть, запах материалов           |
| **G** | Gustatory (вкусовая)          | Чувствует вкус          | Вкус продуктов, напитков                     |

## Справочник: Методы воздействия на конфликт

| Метод                          | Описание                                                                                      |
| ------------------------------ | --------------------------------------------------------------------------------------------- |
| Устранение барьера             | Атрибут полностью убирает причину, из-за которой клиент не может перейти к желаемому действию |
| Снижение интенсивности барьера | Атрибут уменьшает силу барьера, делая переход более приемлемым                                |
| Обход барьера                  | Атрибут позволяет достичь цели, минуя барьер другим путём                                     |
| Рефрейминг барьера             | Атрибут меняет восприятие барьера, делая его несущественным                                   |
| Усиление драйвера              | Атрибут увеличивает привлекательность желаемого состояния                                     |

## Чек-лист проверки заполненной таблицы

Для каждой строки проверьте:

- [ ] Атрибут сформулирован конкретно (не абстрактно)
- [ ] Указана репрезентативная система (не пустая ячейка)
- [ ] Указан тип генерации опыта: (+) появление или (−) исчезновение
- [ ] Опыт описан сенсорно (начинает/перестаёт видеть/слышать/ощущать), а не интерпретативно
- [ ] Связь с конфликтом указывает конкретный элемент грозовой тучи
- [ ] Атрибут отличает артефакт от конкурентов (годится для сравнения)
