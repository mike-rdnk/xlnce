# Инструкция по написанию базового нарратива ДКЦП

## Введение

Базовый нарратив — это связный текст, собранный из аргументов ДКЦП-модели, который становится единой точкой истины (point of truth) для маркетинга, продаж и стратегии. Нарратив объединяет все ключевые элементы ценностного предложения в логически последовательный рассказ.

Для простоты обозначим блоки ДКЦП цифрами и будем использовать их в этой инструкции.

### Нумерация блоков

| Номер | Блок модели                     |
| ----- | ------------------------------- |
| 1     | Клиентский сегмент              |
| 2     | Мотивационный конфликт          |
| 3     | Артефакт                        |
| 4     | Атрибуты артефакта              |
| 5     | Вклад клиента                   |
| 6     | Условия обмена                  |
| 7     | Атрибуты компании               |
| 8     | Ценность                        |
| 9     | Аргументы                       |

## Обязательные аргументы для базового нарратива

Согласно критериям сходимости ДКЦП, базовый нарратив должен содержать все обязательные аргументы. Каждый аргумент связывает два или более элементов модели:

1. **О клиенте и конфликте (1 + 2 → 9)**
   - Проблематизирующий аргумент, который описывает клиентский сегмент и его мотивационный конфликт
   - Формулирует проблему, которую решает ценностное предложение

2. **О конфликте, артефакте и атрибуте артефакта (2 + 3 + 4 → 9)**
   - Аргумент, показывающий, как артефакт и его атрибуты разрешают мотивационный конфликт
   - Демонстрирует связь между проблемой клиента и решением

3. **О клиенте и артефакте (1 + 3 → 9)**
   - Аргумент, связывающий клиентский сегмент с предлагаемым артефактом
   - Показывает соответствие артефакта потребностям сегмента

4. **Об артефакте и материальном вкладе (3 + 5 → 9)**
   - Аргумент, обосновывающий цену артефакта
   - Связывает стоимость с ценностью артефакта

5. **Об условиях обмена и нематериальном вкладе (5 + 6 → 9)**
   - Аргумент, описывающий процесс получения артефакта и необходимые усилия клиента
   - Объясняет, что нужно сделать клиенту для получения решения

6. **О ценности и атрибутах компании (8 + 7 → 9)**
   - Аргумент, показывающий, как атрибуты компании подтверждают заявленные ценности
   - Демонстрирует соответствие компании ценностям клиента

7. **О клиенте и ценности (1 + 8 → 9)**
   - Аргумент, связывающий ценности с клиентским сегментом
   - Показывает, почему эти ценности важны для данного сегмента

8. **Об атрибуте и ценности (4 + 8 → 9)**
   - Аргумент, показывающий, как конкретные атрибуты артефакта поддерживают заявленные ценности
   - Демонстрирует символический смысл атрибутов

## Структура базового нарратива

Базовый нарратив должен быть структурирован логически и последовательно. Рекомендуемый порядок:

### 1. Введение: Проблема клиента
Начните с аргумента о клиенте и конфликте (1 + 2). Это проблематизирующая часть, которая:
- Описывает клиентский сегмент
- Формулирует мотивационный конфликт
- Показывает ущерб от текущего образа действия

### 2. Представление решения: Артефакт и компания
Включите аргументы:
- О клиенте и артефакте (1 + 3)
- О ценности и атрибутах компании (8 + 7)

Эта часть представляет:
- Что именно предлагается (артефакт)
- Кто предлагает (компания) и почему ей можно доверять

### 3. Механизм решения: Как это работает
Включите аргумент о конфликте, артефакте и атрибуте (2 + 3 + 4). Эта часть:
- Показывает, как артефакт и его атрибуты разрешают мотивационный конфликт
- Демонстрирует конкретные свойства решения
- Объясняет механизм работы

### 4. Ценностное обоснование
Включите аргументы:
- Об атрибуте и ценности (4 + 8)
- О клиенте и ценности (1 + 8)

Эта часть:
- Показывает символический смысл решения
- Связывает решение с ценностями клиента
- Объясняет, почему это важно для сегмента

### 5. Условия получения: Вклад и процесс
Включите аргументы:
- Об артефакте и материальном вкладе (3 + 5)
- Об условиях обмена и нематериальном вкладе (5 + 6)

Эта часть:
- Объясняет стоимость решения
- Описывает процесс получения
- Указывает необходимые усилия клиента

### 6. Заключение: Призыв к действию
Завершите нарратив кратким резюме, которое:
- Подчеркивает ключевую ценность
- Мотивирует к действию
- Связывает все элементы воедино

## Правила написания нарратива

### 1. Связность текста
- Аргументы должны быть связаны логическими переходами
- Используйте связующие слова и фразы для плавного перехода между аргументами
- Избегайте резких скачков между темами

### 2. Язык и стиль
- Используйте язык, понятный типичному представителю клиентского сегмента
- Избегайте технического жаргона, если он не понятен клиенту
- Говорите о проблемах клиента, а не о фичах продукта

### 3. Конкретность
- Используйте конкретные формулировки вместо абстрактных
- Указывайте конкретные цифры, сроки, характеристики
- Избегайте общих фраз типа "качественный продукт" или "лучшее решение"

### 4. Последовательность
- Следуйте логической последовательности: проблема → решение → как работает → почему важно → как получить
- Не перескакивайте между блоками без логических связок

### 5. Полнота
- Убедитесь, что включены все обязательные аргументы
- Проверьте, что каждый аргумент связывает минимум два элемента модели
- Не включайте аргументы, которые описывают только один элемент

## Проверка базового нарратива

После написания нарратива проверьте его на соответствие следующим критериям:

### Проверка полноты аргументов
- [ ] Есть аргумент о клиенте и конфликте (1 + 2)
- [ ] Есть аргумент о конфликте, артефакте и атрибуте (2 + 3 + 4)
- [ ] Есть аргумент о клиенте и артефакте (1 + 3)
- [ ] Есть аргумент об артефакте и материальном вкладе (3 + 5)
- [ ] Есть аргумент об условиях обмена и нематериальном вкладе (5 + 6)
- [ ] Есть аргумент о ценности и атрибутах компании (8 + 7)
- [ ] Есть аргумент о клиенте и ценности (1 + 8)
- [ ] Есть аргумент об атрибуте и ценности (4 + 8)

### Проверка качества аргументов
- [ ] Все аргументы представляют собой полные законченные предложения
- [ ] Нет аргументов, связанных менее чем с двумя элементами модели
- [ ] Аргументы не содержат невосстановимых из контекста метамодельных искажений

### Проверка связности
- [ ] Нарратив читается как связный текст, а не набор отдельных утверждений
- [ ] Есть логические переходы между аргументами
- [ ] Структура нарратива соответствует рекомендуемой последовательности

### Проверка языка
- [ ] Язык понятен типичному представителю клиентского сегмента
- [ ] Используется язык проблем клиента, а не язык фич продукта
- [ ] Формулировки конкретны, а не абстрактны

## Пример структуры базового нарратива

```
[Введение: Проблема клиента]
[Аргумент 1 + 2: О клиенте и конфликте]

[Представление решения]
[Аргумент 1 + 3: О клиенте и артефакте]
[Аргумент 8 + 7: О ценности и атрибутах компании]

[Механизм решения]
[Аргумент 2 + 3 + 4: О конфликте, артефакте и атрибуте]

[Ценностное обоснование]
[Аргумент 4 + 8: Об атрибуте и ценности]
[Аргумент 1 + 8: О клиенте и ценности]

[Условия получения]
[Аргумент 3 + 5: Об артефакте и материальном вкладе]
[Аргумент 5 + 6: Об условиях обмена и нематериальном вкладе]

[Заключение: Призыв к действию]
```

## Важные замечания

1. **Нарратив — это не просто список аргументов**, а связный текст, который читается как единое целое.

2. **Порядок аргументов может варьироваться** в зависимости от специфики ценностного предложения и канала коммуникации, но все обязательные аргументы должны присутствовать.

3. **Нарратив может быть адаптирован** под разные каналы (сайт, презентация, коммерческое предложение), но базовая структура и обязательные аргументы должны сохраняться.

4. **Перед написанием нарратива убедитесь**, что модель ДКЦП прошла проверку на сходимость согласно инструкции [[dkcp-checkup-criteria]].

5. **Нарратив должен быть проверен** на соответствие всем критериям сходимости, особенно на сходимость по аргументам.
