# Инструмент рефлексии качества интервью (IQRT) для исследований ДКЦП

## Назначение

IQRT (Interview Quality Reflection Tool) — инструмент систематической рефлексии качества проведённых интервью. Адаптирован для исследований по методологии ДКЦП с целью извлечения качественных данных о мотивационных конфликтах, клиентских сегментах и атрибутах продукта.

Ключевая проблема: качество текста для анализа определяется на этапе интервью. Никакой последующий анализ не компенсирует поверхностные или искажённые данные.

---

## Структура инструмента

IQRT представляет собой таблицу, где:
- **Строки** — транскрибированные вопросы/реплики интервьюера
- **Столбцы** — критерии оценки качества вопроса

### Обязательные столбцы

| Столбец             | Содержание                                                               |
| ------------------- | ------------------------------------------------------------------------ |
| Временна́я метка     | Время в записи                                                           |
| Вопрос/реплика      | Дословная транскрипция того, что сказал интервьюер                       |
| Цель вопроса        | Чего хотел добиться этим вопросом                                        |
| Соответствует цели? | Да/Нет — помог ли вопрос достичь заявленной цели                         |
| Понят респондентом? | Да/Нет/Частично                                                          |
| Сбил с мысли?       | Да/Нет — прервал ли вопрос ход размышлений респондента                   |
| Ввёл новое понятие? | Да/Нет — ввёл ли интервьюер концепт, которого не было в речи респондента |
| Какое понятие ввёл  | Если да — какое именно                                                   |
| Тип вопроса         | См. раздел «Типология вопросов» ниже                                     |
| Открытый/закрытый   | О/З                                                                      |
| Качество ответа     | Да/Нет — устраивает ли качество полученного ответа                       |
| Комментарий         | Свободное поле для рефлексии                                             |

---

## Процедура использования

### Этап 1. Селективная транскрипция

**Если нет транскрипта:**
Прослушать запись интервью. Транскрибировать **только реплики интервьюера** — вопросы, уточнения, комментарии. Ответы респондента не записывать, но держать в контексте при оценке.

**Если транскрипт уже есть:**
Создать таблицу и выписать в неё только вопросы исследователя с таймкодами или номерами строк.

Это позволяет:
- Сфокусироваться на собственных действиях
- Сократить время (полная транскрипция не нужна для этой задачи)
- Оценивать качество интервьюирования независимо от того, насколько «разговорчив» был респондент

### Этап 2. Повторное прослушивание с оценкой

Прослушать запись второй раз. После каждой реплики интервьюера — пауза. Заполнить все столбцы оценки для этой реплики.

Ключевые вопросы для рефлексии:
- Какую цель преследовал этот вопрос?
- Помог ли он получить данные для ДКЦП-модели?
- Не исказил ли я ответ подсказкой или наводящей формулировкой?
- Упустил ли я возможность углубиться в важную тему?

### Этап 3. Анализ паттернов

После заполнения всей таблицы — анализ на уровне целого интервью:
- Какие типы вопросов преобладают?
- Сколько раз вводились новые понятия (не от респондента)?
- Где были упущены возможности для зондирования?
- Какие блоки ДКЦП получили достаточно данных, какие — нет?

## Специфика применения для ДКЦП

### Цели вопросов в контексте ДКЦП

При заполнении столбца «Цель вопроса» привязывайте к блокам модели:

| Цель                                                         | Блок ДКЦП              |
| ------------------------------------------------------------ | ---------------------- |
| Выявить контекст возникновения обмена                        | Клиентский сегмент     |
| Описать текущий или желаемый образ деятельности              | Мотивационный конфликт |
| Понять, что именно удерживает клиента в существующем решении | Барьер / противоречие  |
| Узнать критерии выбора                                       | Атрибуты продукта      |
| Понять, что ценят в решении                                  | Ценность               |

### Типичные ошибки в ДКЦП-интервью

**1. Преждевременное введение терминологии**

Плохо: «Расскажите о вашем мотивационном конфликте при выборе...»

Хорошо: «Вы упомянули, что хотели X, но в итоге не сделали. Расскажите подробнее — что остановило?»

**2. Вопросы о мнениях вместо поведения**

Плохо: «Как вы считаете, важен ли сервис при покупке?»

Хорошо: «Вспомните последнюю покупку [продукт]. Как выбирали? Что проверяли?»

**3. Наводящие вопросы с подсказкой ответа**

Плохо: «Наверное, цена была решающим фактором?»

Хорошо: «Что в итоге определило выбор?»

**4. Переход к следующей теме без зондирования**

Если респондент сказал «было сложно выбрать» — это сигнал к углублению, а не к следующему вопросу из гайда.

### Индикаторы качественных данных для ДКЦП

Хороший вопрос приводит к ответу, где респондент:
- Описывает конкретную ситуацию (время, место, контекст)
- Говорит от первого лица о своём опыте
- Называет конкретные действия («я сделал», «я проверил»)
- Объясняет логику своих решений
- Описывает внутренний конфликт («с одной стороны... с другой»)

Плохой признак — ответ:
- В общих категориях («обычно люди...», «все так делают»)
- Во втором лице («когда ты выбираешь, ты смотришь на...»)
- Декларативный («качество важно», «сервис должен быть хорошим»)

## Типология вопросов

Авторы IQRT ссылаются на классификацию Smith et al. (2022) из книги «Interpretative Phenomenological Analysis: Theory, Method, and Research». Полная типология включает 8 категорий, которые можно разделить на две группы по функции.

### Группа 1: Вопросы о том, ЧТО произошло

| Тип                            | Функция                                                      | Пример для ДКЦП                                                         |
| ------------------------------ | ------------------------------------------------------------ | ----------------------------------------------------------------------- |
| **Описательный (Descriptive)** | Получить детальное описание ситуации, контекста, обстановки  | «Опишите ситуацию, когда вы последний раз выбирали [продукт]»           |
| **Нарративный (Narrative)**    | Получить хронологическую последовательность событий, историю | «Расскажите по шагам, как вы пришли к этому решению — с чего началось?» |
| **Структурный (Structural)**   | Понять организацию опыта, взаимосвязи элементов              | «Кто ещё был вовлечён в это решение? Как распределялись роли?»          |

### Группа 2: Вопросы о том, КАК респондент осмысляет опыт

| Тип                             | Функция                                                      | Пример для ДКЦП                                           |
| ------------------------------- | ------------------------------------------------------------ | --------------------------------------------------------- |
| **Оценочный (Evaluative)**      | Понять оценку, отношение, значимость события для респондента | «Что было самым сложным в этой ситуации?»                 |
| **Сравнительный (Comparative)** | Сопоставить опыт с другими ситуациями, выявить различия      | «Чем этот выбор отличался от предыдущего?»                |
| **Циркулярный (Circular)**      | Понять, как респондент видит восприятие других людей         | «Как, по вашему мнению, это выглядело со стороны коллег?» |

### Группа 3: Технические приёмы

| Тип                      | Функция                                                           | Пример                                           |
| ------------------------ | ----------------------------------------------------------------- | ------------------------------------------------ |
| **Промпт (Prompt)**      | Короткий стимул для продолжения («и?», «а дальше?», кивок, пауза) | «Угу...», «И что потом?»                         |
| **Проба / зонд (Probe)** | Углубление в уже сказанное, запрос деталей                        | «Вы сказали "было неудобно" — можете подробнее?» |


### Баланс типов вопросов

Ключевой принцип: нужен баланс между вопросами о фактах (что произошло) и вопросами об осмыслении (как понимает).

**Для ДКЦП критично:**
- Начинать с описательных и нарративных вопросов — получить фактуру
- Использовать идиографические зонды (I) — от обобщений к конкретике
- Переходить к оценочным и объяснительным — понять логику выбора
- Сравнительные вопросы — для выявления критериев и атрибутов

**Типичная ошибка:** слишком рано переходить к объяснительным вопросам («почему?»), не получив достаточно описательного материала. Респондент начинает рационализировать вместо того, чтобы вспоминать.

### Применение в столбце «Тип вопроса»

При заполнении IQRT используйте сокращения:

- **Оп** — описательный
- **Нр** — нарративный
- **Ст** — структурный
- **Оц** — оценочный
- **Ср** — сравнительный
- **Цр** — циркулярный
- **Пр** — промпт
- **Зн** — зонд

## Режим использования

### Для обучения интервьюеров

1. Провести интервью
2. Заполнить IQRT самостоятельно
3. Обсудить с наставником конкретные вопросы
4. Определить 2-3 навыка для развития
5. Повторить цикл

### Для самоконтроля опытного исследователя

Применять выборочно — к каждому 3-5 интервью или к проблемным случаям (когда данные кажутся «пустыми»).

### Для аудита исследовательского проекта

Руководитель проекта анализирует IQRT нескольких интервью разных интервьюеров для калибровки подходов.

---

## Ограничения инструмента

IQRT оценивает только действия интервьюера. Он не заменяет:
- Анализ качества самих данных
- Оценку репрезентативности выборки
- Проверку валидности интерпретаций

Качественное интервью — необходимое, но не достаточное условие качественного исследования.
